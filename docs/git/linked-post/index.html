Git심화 과정 익혀보기 및 꼭 외워야할 명령어 모음집!

By 이건영 2023년 5월 29일

GIT :

menu = "main"

add, commit, push, pull

show, status, log, shortlog, diff

clone, checkout, merge, fetch, rebase

reset, statsh, blame, cherry-pick, submodule, tag

​

지금부터 집중해서 화이팅!

init

.git 이라는 파일 생성

clone

github에 있는 저장소(기존 저장소)를 복제

​

자동으로 로컬의 master 브랜치가 리모트 저장소의 master 브랜치를 추적하도록 함.

checkout

* 수정된 파일 되돌리기

git checkout -- [file]

​

매우 위험하므로 사용하지 않는 것을 추천

​

* 브랜치 체크아웃

git checkout -b [체크아웃할 브랜치 이름]

add

Stage 상태로 파일을 올리는 것.

Stage 상태로 올라가면 파일이 추적됨.

branch

* 브랜치 목록을 보여줌

git branch

​

브랜치 목록 + 브랜치마다 마지막 커밋 메세지 함께 보기

* git branch -v

​

* 현재 checkout한 브랜치 기준으로 merged/no-merged된 브랜치인지 필터링해서 볼 수 있음

git branch --merged

git branch --no-merged

​

reset

stash

commit

Staging Area에 속한 스냅샷을 커밋함.

​

Unstaged 상태의 파일은 커밋되지 않는다. (= add를 먼저 해야한다)

커밋하기 전에 git status로 변경사항 모든 것이 Staged 상태인지 확인 할 수 있다.

​

* add + commit을 한번에 하는 옵션

git commit -a

​

* 메세지 인라인 첨부

git commit -m

​

* 커밋 되돌리기

git commit --amend

push

remote 저장소에 데이터를 올림

​

git push [remote 저장소 이름] [브랜치 이름]

​

오픈소스 프로젝트의 경우, contribute 하고 싶을 경우 Upstream 저장소에 Push 할 수 있다.

pull

fetch + merge

리모트 저장소 브랜치에서 데이터를 가져올 뿐만 아니라 자동으로 로컬 브랜치와 Merge 시킴

merge

합쳐.. merge

fetch

로컬저장소에는 없지만 리모트 저장소에 있는 데이터를 가져옴

git fetch [remote 저장소 이름]

rebase

Base가 갱신이 필요한 경우 사용

​

0) diff 저장 1) Rewind 2)Rebase 3)merge --> 해당 키워드는 내가 지어낸 것임

이렇게 3단계로 이루어 진 것을 한번에 하는 것이 rebase

show

remote

현재 프로젝트에 등록된 리모트 저장소 확인

​

* remote 추가하기

git remote add [저장소이름] [URL]

​

* url 과 함께 보기

git remote -v

​

* 특정 리모트 저장소 살펴보기 (저장소의 구체적인 정보 확인 가능)

git remote show [remote 저장소 이름]

​

* remote 저장소 이름 변경

git remote rename [현재 이름] [바꿀 이름]

​

* remote 저장소 삭제

git remote remove [remote 저장소 이름]

git remote rm [remote 저장소 이름]

status

현재 어떤 브랜치에 있는지

현재 브랜치의 상태

커밋 상태

​

* 변경 상태 짤막하게 확인하기

git status -s

git status --short

diff

status와 다른점은 아직 staged 상태가 아닌 파일을 비교해 볼 수 있고,

어떤 라인을 추가하고, 삭제했는지 자세하게 확인이 가능하다.

(= git add 하지 않고도 변경사항을 확인 할 수 있음)

(= git add 후에는 변경 상태를 보여주지 않는다.)

​

* 저장소에 커밋한것과 Staging Area에 있는 것을 비교해서 보여줌.

git diff --stated

(= git diff --cached)

log

커밋 로그들 표시

​

commit id

Author:

Date:

커밋 메세지

​

* 각 커밋의 diff 표시

git log -p

git log --patch

shortlog

커밋 로그들 간단히 보기 위함.

​

commit id는 표시 안됨.

blame

누가 어떤부분을 수정했는지 확인
