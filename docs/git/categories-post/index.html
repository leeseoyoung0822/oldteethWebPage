Git에 대해 알아보자!

By 이건영 5월 28일, 2023

지금부터 나랑 함꼐 Git에 대해 제대로 배워보자!

개발자라면 알아야할 필수 GIT명령어!

지금부터는 모르면 안되는 필수명령어부터 일단 말할께!

  • 깃 시작-git init
  • git add '파일이름'

    • 로컬 저장소(= working directory, working tree)에 저장된 파일을 staging area(= cashe, index)로 전달 ( 인자로 . 를 전달하면 로컬 저장소 안의 모든 파일을 area로 전달)

  • 깃 사용내역 보여주기-git log
  • git commit -옵션 "#버전이름 ":

    staging area의 수정된 파일들을 repo로 전달, 옵션을 통해 버전 관리 가능

commit에 관련된 추가적인 알려주고 싶은거!

-m : 별도의 메세지를 작성할 필요없이 인라인 형식으로 바로 커밋 메세지(=버전 이름) 작성.

-a : 별도의 add명령어를 사용하지 않고 수정된 파일에 대해 add, commit을 한번에 수행함. (단, 한번도 add되지 않은 파일은 add를 따로 작업 해줘야함.)

-am : a, m의 옵션을 합친 형태.

그럼 깃은 누가써야해?

  • 팀프로젝트를 진행하고 싶은 사람들
  • 나의 프로젝트를 저장해놓고 싶은 사람들
  • 그냥 모든 개발자들...?

    • Git에 대한 정보 한눈에 알아보기!
